# Narrative Memory Protocol

## Purpose
Give the agents persistent narrative memory so they do not misread recursive structure as isolated local effect.

## Required Memory Files

### character_models.md
For each major character:
- initial presentation
- reader assumptions encouraged by the book
- later complications
- final understanding
- contradictions preserved
- contradictions resolved
- risk of flattening or over-explanation

### relationship_arcs.md
For each major relationship:
- starting dynamic
- pressure points
- reversals
- misunderstandings
- intimacy mechanisms
- final state
- emotional payoff quality

### motif_index.md
For each recurring motif, image, phrase, object, joke, setting, or behavioral pattern:
- first appearance
- later returns
- transformation of meaning
- payoff strength
- whether the motif feels organic or over-signaled

### structural_map.md
Map:
- parts/acts
- turning points
- delayed revelations
- escalation patterns
- quiet chapters with later payoff
- chapters that remain structurally unsupported

### retrospective_recontextualization_log.md
Track:
- scenes whose meaning changes after later events
- early lines that become more powerful later
- moments that invite misreading intentionally
- moments that appear accidental rather than designed

### unresolved_questions.md
Track:
- genuine productive ambiguity
- possible confusion
- holes in motivation
- emotional gaps
- world/context gaps

## Rule
Memory files are not final criticism. They are the evidence base for criticism.
