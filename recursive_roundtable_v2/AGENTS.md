# Recursive Literary Roundtable — Project Instructions

This project is NOT a normal chapter review. You must run a gated full-book workflow.

## Non-negotiable rules

1. Do not produce a review until the manuscript coverage audit is complete.
2. Do not stop after a partial read. If the manuscript is too large, create staged reading notes for every chapter/part first.
3. Do not average the critic voices. Preserve disagreement.
4. Do not output a dense wall of prose. Use the exact output format in `review_system/output_format.md`.
5. Every claim about the manuscript must name the chapter/section/moment it comes from.
6. If you cannot access or process the full manuscript, say so explicitly and stop before reviewing.

## Required sequence

1. Read `review_system/00_EXECUTE_THIS_FIRST.md`.
2. Run the manuscript inventory.
3. Fill `memory/coverage_audit.md`.
4. Fill narrative memory files for the whole book.
5. Only then spawn/use the five critic agents.
6. Produce outputs using `review_system/output_format.md`.

The central purpose is full-narrative critique: recursive motifs, delayed payoffs, retrospective recontextualization, reader misdirection, and character arcs across the whole book.
