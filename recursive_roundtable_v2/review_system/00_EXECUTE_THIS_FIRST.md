# Execute This First

You are running a whole-book recursive literary review system.

## Critical failure prevention

Before any criticism, you must prove full manuscript coverage.

Create a table with one row per chapter/section:

| Unit | Heading / file | Read status | 3-5 sentence neutral summary | Key callbacks/motifs | Characters updated |

You may not write the final review until every row says `Read`.

If the manuscript is too long for one pass:

- Split into parts.
- Read each part.
- Write neutral memory notes after each part.
- Continue until all parts are covered.
- Then begin criticism.

## Required command to yourself

Use this exact operating rule:

> I will not review the book until I have produced a complete coverage audit and narrative memory across the entire manuscript.

## Manuscript target

Preferred file:

`book/manuscript.md`

If the manuscript is split across files, inventory every file in `/book/` and process them in order.
